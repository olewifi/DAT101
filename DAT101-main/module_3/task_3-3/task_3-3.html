1. What is the difference between invoking and creating a function?

    2. Creating a function means that you are giving the function instructions to execute when called upon. Invoking the function is calling the function and it begins to execute the instructions that has been given. The difference is that creating does not mean the code is being executed, while invoking it does.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons helps the program understand when a statement ends and the next one begins. Without semicolons the program will have a hard time to understand what is written. We can compare this to our written language on how multiple sentences would have looked like without periods. Code blocks groups statements together into a single unit. The syntax here is that the program will read them together and not separately. This is convenient when writing, and using functions, loops, or conditional coding.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. It will return as "Undefined".

  4. How do we separate parameters from each other when creating a function?

    5. We separate parameters by using commas and have them within parentheses.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. When creating functions we want specific parameters sometimes, so by matching the parameter names to what we want makes it easier during the invocation to put in the right input.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. When the parameter is missing it is not defined which makes it return as undefined.
